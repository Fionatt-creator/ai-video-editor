# LifeAnchor 产品路线图 (Roadmap)

> 版本：v1.0  
> 更新日期：2026-03-26  
> 目标：整合标准化量表 + 五大维度 + 日常工具，构建获客-留存-增长的完整闭环

---

## 一、核心产品架构

### 1.1 理论基础：五大核心维度

| 维度 | 英文名 | 核心指标 | 对应量表 |
|------|--------|----------|----------|
| 情绪幸福感 | Hedonic Well-being | 日常情绪波动、积极体验频率 | PANAS |
| 认知评价 | Life Evaluation | 生活满意度、整体评价 | SWLS |
| 心理幸福感 | Psychological Well-being | 自我接纳、个人成长、自主性 | OHQ |
| 社会幸福感 | Social Well-being | 社会关系质量、归属感 | 自定义量表 |
| 意义与目的感 | Eudaimonic Well-being | 人生意义、目标清晰度 | LifeAnchor 核心对话 |

### 1.2 量表整合策略

| 量表 | 用途 | 使用场景 |
|------|------|----------|
| **SWLS** (生活满意度量表) | 基线评估 + 长期追踪 | 注册时、每月回顾 |
| **OHQ** (牛津幸福感问卷) | 心理幸福感深度测量 | 季度评估 |
| **PANAS** (积极/消极情感量表) | 日常情绪追踪 | 每周自测、情绪异常时 |

## 二、功能路线图

### Phase 1: 基础评估与获客 (MVP+1-2个月)

**核心目标：用「免费自我测试」作为获客钩子，建立初步用户池**

#### 功能 1.1: 五分钟幸福自测（获客入口）

**产品形态：**
- 移动端友好的问卷界面
- 整合 SWLS + PANAS 精选题目（共10-15题）
- AI 即时生成个性化报告

**使用流程：**
```
扫码/点击 → 5分钟问卷 → AI分析报告 → 预约完整版 → 下载App
```

**报告内容：**
1. 五维度雷达图（可视化当前状态）
2. 关键洞察（"你的情绪幸福感高于78%的人，但意义感偏低"）
3. 一个具体行动建议（"试试今晚写下今天最感激的3件事"）
4. 引导下载完整版 App（"想获得完整的人生地图？"）

**获客钩子设计：**
- **场景化文案**："感到空虚却不知道原因？5分钟找到你的人生短板"
- **社交分享**：报告页一键生成分享图（适合小红书/朋友圈）
- **稀缺性**："前1000名免费解锁完整版人生地图"

---

#### 功能 1.2: 人生意义地图 1.0

**核心流程：**
1. **价值观探索**（基于 ACT 疗法）
2. **热情领域识别**（基于 OHQ 维度）
3. **人生目标澄清**（引导式对话）
4. **微行动推荐**（基于当前情绪状态 PANAS）

**可视化输出：**
```
         [核心价值观] ← 意义与目的感
              |
    [热情领域]——[人生目标]——[日常行动]
         ↑              |           ↓
    心理幸福感      认知评价      情绪幸福感
              |
         [支持系统] ← 社会幸福感
```

---

### Phase 2: 日常习惯养成 (第3-4个月)

**核心目标：提升留存，建立使用习惯**

#### 功能 2.1: 幸福日记（每日追踪）

**产品形态：**
- 每日推送（可选择时间：早/中/晚）
- 3分钟完成，支持语音输入
- AI 自动提取关键情绪标记

**内容结构：**
1. **今日高光**（1件积极事件）→ 训练积极关注
2. **情绪温度计**（1-10分）→ PANAS 简化版
3. **意义时刻**（今天什么让你感到"值得"）→ 意义感追踪
4. **感恩记录**（1-3件感恩的事）→ 认知评价提升

**智能功能：**
- 情绪异常自动提醒（连续3天低于5分 → 推送关怀内容）
- 周/月自动总结（生成个人情绪趋势图）
- AI 个性化反馈（"这周你的情绪比上周稳定，继续保持"）

---

#### 功能 2.2: 周回顾（PERMA模型）

**每周日推送，基于 Seligman 的 PERMA 框架：**

| 维度 | 回顾问题 | 数据支撑 |
|------|----------|----------|
| **P**ositive emotion | 这周让你最开心的事？ | 幸福日记聚合 |
| **E**ngagement | 什么时候你感到"心流"？ | 用户自评 |
| **R**elationships | 这周和谁有深度连接？ | 社会幸福感追踪 |
| **M**eaning | 这周做了什么有意义的事？ | 意义感追踪 |
| **A**ccomplishment | 这周完成了什么目标？ | 微行动完成率 |

**输出：**
- 本周五维度得分变化
- 下周建议关注维度
- 与上周/上月对比趋势

---

#### 功能 2.3: 身体信号法（创新功能）

**原理：** 情绪会在身体上有表现（紧绷、疲惫、 energized）

**使用流程：**
1. **身体扫描引导**（3分钟语音引导）
2. **标记身体感受**（示意图选择：肩膀紧绷、胃部不适等）
3. **AI 关联分析**（"你提到肩膀紧绷，这周有工作压力吗？"）
4. **身心建议**（对应放松练习、运动建议）

**数据价值：**
- 建立"情绪-身体"关联数据库
- 预警功能：身体信号异常 + 情绪低分 → 推送深度关怀

---

### Phase 3: 深度干预与留存 (第5-6个月)

**核心目标：提升付费转化，建立长期价值**

#### 功能 3.1: 精准干预模块

**触发条件：**
| 触发信号 | 干预内容 |
|----------|----------|
| SWLS 连续下降 | 启动"生活满意度提升计划"（4周课程） |
| PANAS 消极情绪持续偏高 | 情绪调节技巧训练（CBT 基础） |
| 意义感得分低于30% | "寻找意义"深度工作坊 |
| 社会幸福感低 | 社群匹配/关系建设指导 |

**形式：**
- 结构化课程（2-4周）
- AI + 真人教练结合
- 进度追踪与成就系统

---

#### 功能 3.2: 社群与同伴支持

**基于五维度的社群匹配：**
- "意义探索者"社群（意义感偏低但成长意愿强）
- "情绪管理"互助组（PANAS 异常用户）
- "价值观澄清"工作坊（定期线上活动）

---

### Phase 4: 数据智能与个性化 (第7-12个月)

**核心目标：利用数据构建护城河**

#### 功能 4.1: 个人幸福预测模型

基于用户长期数据，预测：
- 下周情绪趋势
- 哪些因素最影响你的幸福感
- 最佳行动时机（个性化推送）

#### 功能 4.2: 群体洞察报告

**匿名化数据输出：**
- "28岁产品经理群体的幸福画像"
- "不同城市的幸福感差异"
- "职业与意义感的相关性"

**价值：** 内容营销素材、媒体传播点

## 三、获客策略：自我测试作为增长引擎

### 3.1 免费自我测试设计

#### 主测试：「你的幸福短板在哪里？」

**题目设计（精选10-15题）：**
- 3题 SWLS 核心题（生活满意度）
- 4题 PANAS 精简版（情绪状态）
- 3题 OHQ 核心维度（心理幸福感）
- 2-3题 原创题（意义感初筛）

**结果页设计：**
```
┌─────────────────────────────────────────┐
│  你的幸福五维图                          │
│  [雷达图：情绪↑ 意义↓ 心理↑ 认知→ 社会↑] │
├─────────────────────────────────────────┤
│  关键发现：                              │
│  "你的情绪调节能力很强，但正在经历        │
│   意义感缺失——这是现代人的通病"          │
├─────────────────────────────────────────┤
│  立即行动：                              │
│  试试这个2分钟练习 → [开始练习]          │
├─────────────────────────────────────────┤
│  [生成我的分享图]  [下载App解锁完整版]    │
└─────────────────────────────────────────┘
```

#### 子测试矩阵（针对不同获客场景）

| 测试名称 | 目标人群 | 获客场景 |
|----------|----------|----------|
| 「你是否正在经历 quarter-life crisis？」 | 25-30岁职场人 | 小红书、即刻 |
| 「测测你的'职业倦怠'指数」 | 互联网从业者 | 脉脉、LinkedIn |
| 「你的'空心病'有多严重？」 | 高知群体 | 知乎、公众号 |
| 「30秒找到你的核心驱动力」 | 通用 | 朋友圈裂变 |

---

### 3.2 获客渠道与内容策略

#### 渠道 1: 社交媒体（小红书 + 即刻）

**内容类型：**
- **结果分享图**：用户分享自己的五维图（病毒式传播）
- **维度科普**："为什么物质丰富却不幸福？可能是这个维度出了问题"
- **用户故事**："做完这个测试，我发现自己一直在逃避真正在意的事"

**KOL 合作：**
- 心理健康博主
- 职场成长博主
- 生活方式博主

#### 渠道 2: 职场社区（脉脉 + LinkedIn）

**内容角度：**
- "年薪50万但不想上班？测测你的职业倦怠指数"
- "35岁危机的本质是意义感危机"
- "大厂人为什么普遍空虚？数据告诉你真相"

#### 渠道 3: 内容平台（知乎 + 公众号）

**深度内容：**
- 《基于10万份数据的都市白领幸福报告》
- 《SWLS量表解读：为什么你的生活满意度不高？》
- 《从心理学角度看"躺平"：真的是逃避吗？》

#### 渠道 4: 线下场景（活动 + 企业合作）

**快闪测试站：**
- 设置于咖啡馆、书店、联合办公空间
- 扫码→5分钟测试→当场打印"个人幸福卡片"

**企业 EAP 合作：**
- 作为员工心理健康评估工具
- B2B2C 获客模式

---

### 3.3 转化漏斗设计

```
曝光（社交媒体/内容/广告）
    ↓ 10-15%
点击测试链接
    ↓ 40-60%
完成5分钟测试
    ↓ 30-40%
查看完整报告
    ↓ 15-25%
下载 App
    ↓ 40-50%
完成首次对话
    ↓ 30-40%
7日留存
    ↓ 20-30%
付费订阅/深度使用
```

**关键优化点：**
1. **测试完成率**：简化题目，进度条可视化
2. **报告吸引力**：即时性 + 个性化 + 可分享性
3. **App 下载动机**："解锁完整人生地图" + 限时免费
4. **首次体验**：AI 对话必须在5分钟内创造"被理解"的瞬间

## 四、用户触达机制：在正确的时间出现

### 4.1 典型用户画像与触达时机

#### 用户画像 A：「深夜emo型」

**画像：**
- 28岁，互联网从业者
- 经常凌晨1-2点还在刷手机
- 发过"也不知道自己在干嘛"类朋友圈

**触达时机：**
| 时机 | 触达方式 | 内容 |
|------|----------|------|
| 深夜 23:00-01:00 | 朋友圈广告 | "这么晚还没睡，是不是也在想人生意义？" |
| 周日晚上 | 公众号推送 | "明天又要上班了，这份'抗emo指南'收好" |
| 春节/长假后 | 内容营销 | "假期综合征？其实是意义感缺失" |

**转化路径：**
深夜广告 → 测试 → 发现"意义感偏低" → 下载 App → AI 对话 → 获得即时安慰

---

#### 用户画像 B：「职业倦怠型」

**画像：**
- 32岁，中高层管理者
- 工作稳定但缺乏激情
- 经常搜索"转行""副业""中年危机"

**触达时机：**
| 时机 | 触达方式 | 内容 |
|------|----------|------|
| 金三银四/金九银十 | LinkedIn 广告 | "想跳槽？先搞清楚你真正想要什么" |
| 年底绩效季 | 企业内推 | "年终总结不只是给老板的，也是给自己的" |
| 长假前 | 内容营销 | "为什么度假也无法消除你的疲惫？" |

**转化路径：**
LinkedIn 测试 → 发现"价值观与工作冲突" → 深度报告 → 企业版试用

---

#### 用户画像 C：「成长焦虑型」

**画像：**
- 26岁，刚工作2-3年
- 报了各种课程但越学越焦虑
- 担心"落后同龄人"

**触达时机：**
| 时机 | 触达方式 | 内容 |
|------|----------|------|
| 知识付费节后 | 小红书内容 | "买了10门课还是焦虑？问题不在知识" |
| 同龄人晒成就时 | 朋友圈广告 | "别人都在进步，我在原地焦虑？" |
| 新年/生日 | 邮件/推送 | "新的一年，从找到真正的自己开始" |

**转化路径：**
小红书内容 → 测试 → 发现"自我接纳度低" → 下载 App → 21天成长计划

### 4.2 良性循环设计

```
        ┌─────────────────────────────────────┐
        │                                     │
        ↓                                     │
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   获客阶段     │ → │   激活阶段     │ → │   留存阶段     │
│               │    │               │    │               │
│ • 免费测试     │    │ • 首测报告     │    │ • 每日日记     │
│ • 社交分享     │    │ • AI 对话      │    │ • 周回顾      │
│ • 内容营销     │    │ • 意义地图     │    │ • 月度量表     │
└───────────────┘    └───────────────┘    └───────────────┘
        ↑                                     │
        │                                     ↓
        │                              ┌───────────────┐
        │                              │   传播阶段     │
        │                              │               │
        └──────────────────────────────│ • 分享报告     │
                                       │ • 邀请好友     │
                                       │ • 社群互动     │
                                       └───────────────┘
```

**循环关键点：**

1. **分享机制：**
   - 测试报告一键生成分享图
   - 里程碑成就（"连续记录30天"）可分享
   - 邀请好友双方解锁高级功能

2. **数据沉淀：**
   - 用户记录越多，报告越有价值
   - 历史数据对比形成"成长叙事"
   - 增加迁移成本，提升留存

3. **社区效应：**
   - 相似用户匹配（"你们都在探索创造力"）
   - 匿名分享真实故事
   - 群体挑战（"21天感恩日记打卡"）

## 五、关键指标与里程碑

### 5.1 北极星指标

**核心指标：** 「周活跃用户中完成至少一次有意义对话的比例」

**支撑指标：**
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 测试完成率 | >50% | 优化题目设计 |
| 测试→下载转化率 | >20% | 优化报告吸引力 |
| 次日留存率 | >40% | 首次体验质量 |
| 7日留存率 | >25% | 习惯养成 |
| 30日留存率 | >15% | 长期价值验证 |
| NPS | >40 | 用户满意度 |
| 付费转化率 | >5% | 商业化验证 |

### 5.2 里程碑规划

| 阶段 | 时间 | 核心目标 | 关键动作 |
|------|------|----------|----------|
| **MVP** | 1-2个月 | 验证核心假设 | 完成基础测试 + 首次对话功能 |
| **增长** | 3-4个月 | 获客验证 | 测试获客渠道，优化转化率 |
| **留存** | 5-6个月 | 习惯养成 | 上线日记 + 周回顾，提升留存 |
| **变现** | 7-9个月 | 商业模式 | 推出付费深度干预模块 |
| **规模化** | 10-12个月 | 市场扩张 | 企业版、多语言、国际化 |

## 六、风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 用户觉得测试"不准" | 中 | 高 | 强调"自我探索工具"而非"诊断"，增加解释性内容 |
| 隐私担忧 | 中 | 高 | 强调数据本地存储、匿名化处理、用户可控 |
| 留存率低于预期 | 高 | 高 | 增加游戏化元素、社交功能、真人教练支持 |
| 竞品模仿 | 高 | 中 | 建立数据护城河、品牌差异化、用户社区粘性 |
| 监管风险（心理健康类App） | 低 | 高 | 明确非医疗定位，必要时引入专业背书 |
