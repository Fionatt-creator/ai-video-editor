# 小红书运营助手 - 产品需求文档 (PRD)

> 产品目标：自动化监控小红书平台中与视频翻译/AIGC相关的内容动态，辅助运营决策与互动

---

## 一、产品概述

### 1.1 产品定位
个人小红书运营辅助工具，通过自动化监控、内容抓取、互动辅助，帮助运营者：
- 及时掌握行业热点与竞品动态
- 发现潜在爆款内容灵感
- 高效完成用户互动

### 1.2 目标用户
- **用户类型**：个人运营者
- **使用场景**：每日查看运营数据、发现爆款内容、与潜在用户互动

### 1.3 核心痛点
1. 手动搜索效率低，容易错过热点
2. 爆款内容识别标准不清晰，凭感觉判断
3. 评论区互动耗时，想回复但不知道怎么写

## 二、功能模块

### 模块一：搜索词热度监控

#### 2.1.1 功能描述
自动监控小红书平台指定搜索词的搜索结果数量变化，每日汇总到飞书多维表格，追踪热度趋势。

#### 2.1.2 监控关键词清单

| 优先级 | 关键词 | 关联逻辑 |
|--------|--------|----------|
| P0 | 视频翻译 | 核心关键词，直接对应产品能力 |
| P1 | AI配音 | 功能细分，vozo核心功能之一 |
| P1 | 视频重述 | vozo 主要卖点（AI Rewind） |
| P1 | 视频克隆 | vozo 声音/画面克隆功能 |
| P2 | AI视频 | 行业大词，流量入口 |
| P2 | 视频翻译神器 | 长尾词，精准需求 |
| P2 | 短视频翻译 | 场景细分，区分长视频 |
| P3 | 一键翻译视频 | 功能导向词 |
| P3 | 视频多语言 | 需求场景词 |

#### 2.1.3 数据采集字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 日期 | 日期 | 数据采集日期 |
| 关键词 | 文本 | 搜索词 |
| 搜索结果数 | 数字 | 小红书返回的笔记数量 |
| 较前日变化 | 数字 | 环比变化量 |
| 趋势 | 单选 | ↑上升 / ↓下降 / →持平 |
| 采集时间 | 时间 | 当日具体采集时间点 |

#### 2.1.4 采集规则
- **采集频率**：每日一次（建议早上9:00）
- **采集方式**：模拟搜索请求，获取搜索结果总数
- **数据去重**：同一关键词同一日期只保留一条记录

---

### 模块二：爆款内容抓取

#### 2.2.1 功能描述
根据监控到的关键词，抓取小红书相关帖子的关键数据，自动识别爆款（点赞≥50 或 评论≥10），汇总到飞书多维表格。

#### 2.2.2 爆款判定标准

| 指标 | 阈值 | 逻辑关系 |
|------|------|----------|
| 点赞数 | ≥ 50 | OR |
| 评论数 | ≥ 10 | OR |

#### 2.2.3 数据采集字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 采集日期 | 日期 | 抓取时间 |
| 关键词 | 文本 | 触发抓取的关键词 |
| 帖子链接 | 超链接 | 小红书笔记URL |
| 标题 | 文本 | 笔记标题 |
| 作者 | 文本 | 博主昵称 |
| 作者ID | 文本 | 小红书用户ID |
| 点赞数 | 数字 | 当前点赞数量 |
| 收藏数 | 数字 | 当前收藏数量 |
| 评论数 | 数字 | 当前评论数量 |
| 是否爆款 | 复选框 | 自动判定结果 |
| 内容摘要 | 文本 | 笔记正文前100字 |
| 标签 | 多选 | 笔记携带的话题标签 |
| 发布时间 | 日期时间 | 原帖发布时间 |

#### 2.2.4 抓取规则
- **抓取范围**：按关键词搜索，抓取前50条结果
- **去重逻辑**：基于帖子链接去重，已抓取帖子不再重复采集
- **频率控制**：每日每个关键词抓取一次
- **爆款标记**：自动勾选"是否爆款"字段

### 模块三：互动助手（评论抓取+回复生成）

#### 2.3.1 功能描述
针对已抓取的爆款帖子，自动抓取评论区内容，基于上下文生成回复建议，推送至飞书供人工审核后，在小红书App手动发送。

#### 2.3.2 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 抓取评论区                                          │
│  ↓ 针对"是否爆款"=true 的帖子                                │
├─────────────────────────────────────────────────────────────┤
│  Step 2: 智能筛选高价值评论                                   │
│  ↓ 筛选条件：提问/求推荐/表达需求/询问产品                    │
├─────────────────────────────────────────────────────────────┤
│  Step 3: 生成回复建议                                         │
│  ↓ 基于评论内容+帖子主题+产品卖点生成                         │
├─────────────────────────────────────────────────────────────┤
│  Step 4: 推送至飞书多维表格                                   │
│  ↓ 包含：评论原文、回复建议、原帖链接                         │
├─────────────────────────────────────────────────────────────┤
│  Step 5: 人工审核（小红书App操作）                            │
│  ↓ 运营者查看建议，决定是否回复及修改内容                     │
└─────────────────────────────────────────────────────────────┘
```

#### 2.3.3 数据采集字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 采集日期 | 日期 | 抓取时间 |
| 原帖链接 | 超链接 | 被评论的帖子URL |
| 原帖标题 | 文本 | 帖子标题 |
| 评论用户 | 文本 | 评论者昵称 |
| 评论内容 | 文本 | 完整评论原文 |
| 评论点赞数 | 数字 | 该评论的点赞数 |
| 评论类型 | 单选 | 提问/求推荐/表达需求/普通互动 |
| AI回复建议 | 文本 | 系统生成的回复内容 |
| 是否已处理 | 复选框 | 人工标记处理状态 |
| 处理时间 | 日期时间 | 人工操作时间 |

#### 2.3.4 高价值评论识别规则

自动筛选以下类型的评论：

| 类型 | 关键词/句式示例 |
|------|-----------------|
| 提问型 | "怎么做的"、"用什么工具"、"怎么做"、"能教吗" |
| 求推荐型 | "求推荐"、"有推荐的吗"、"用什么软件" |
| 需求表达型 | "我也需要"、"正好在找"、"求教程" |
| 竞品询问型 | "和XX比怎么样"、"哪个更好" |

#### 2.3.5 回复生成策略

**Prompt模板方向**：
- 角色：友善的AIGC爱好者/从业者
- 语气：真诚、有经验但不推销
- 策略：软性种草，提供价值，引导私信
- 字数：控制在50-100字

**回复示例框架**：
```
【场景：有人在问"视频怎么翻译的"】
回复建议：
"我用的是AI翻译工具，可以自动识别语音+生成字幕~
英语日语都能翻，还挺方便的。
如果你有需要可以私信我，分享给你"
```

#### 2.3.6 人工操作流程
1. 每日查看飞书多维表格中的"待处理评论"
2. 点击"原帖链接"跳转至小红书App对应帖子
3. 找到对应评论，复制"AI回复建议"或自行修改
4. 在小红书App发送回复
5. 回到飞书表格，勾选"是否已处理"

## 三、技术方案

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     小红书运营助手                           │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  搜索监控    │  │  内容抓取    │  │  互动助手    │      │
│  │  定时任务    │  │  定时任务    │  │  定时任务    │      │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘      │
│         │                 │                 │              │
│         ▼                 ▼                 ▼              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              小红书数据采集模块                      │   │
│  │  (Cookie登录 + API/爬虫 + 反爬策略)                  │   │
│  └─────────────────────────────────────────────────────┘   │
│         │                 │                 │              │
│         ▼                 ▼                 ▼              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              飞书多维表格写入模块                    │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 小红书数据采集

#### 3.2.1 登录方式
- **方式**：Cookie登录（需要用户提供小红书账号的Cookie）
- **存储**：加密存储Cookie，定期更新

#### 3.2.2 主要接口

| 功能 | 接口/方法 | 说明 |
|------|-----------|------|
| 搜索 | /api/sns/web/v1/search/notes | 获取搜索结果列表 |
| 笔记详情 | /api/sns/web/v1/feed | 获取笔记详细信息 |
| 评论列表 | /api/sns/web/v1/comment/list | 获取笔记评论 |

#### 3.2.3 反爬策略
- 请求间隔：随机1-3秒
- User-Agent轮换
- 失败重试：最多3次，指数退避
- 每日请求量控制

### 3.3 数据存储（飞书多维表格）

建议创建 **3 个数据表**：

1. **搜索词监控表** - 存储模块一数据
2. **爆款内容表** - 存储模块二数据  
3. **互动任务表** - 存储模块三数据

### 3.4 定时任务

| 任务 | 频率 | 时间 | 说明 |
|------|------|------|------|
| 搜索词监控 | 每日 | 09:00 | 采集所有关键词数据 |
| 内容抓取 | 每日 | 09:30 | 抓取前一日到当日的爆款内容 |
| 评论抓取 | 每日 | 10:00 | 抓取爆款帖子的最新评论 |

## 四、MVP 范围（第一版）

### 4.1 功能优先级

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 搜索词监控（核心词） | P0 | 先做"视频翻译"1个词 |
| 爆款内容抓取 | P0 | 抓取+自动判定 |
| 评论抓取 | P1 | 仅抓取，不生成回复 |
| 回复生成 | P2 | 接入AI生成建议 |
| 多关键词支持 | P2 | 扩展至完整关键词清单 |

### 4.2 第一版交付标准

- [ ] 支持1个核心关键词（视频翻译）的每日监控
- [ ] 支持自动识别爆款（点赞≥50 or 评论≥10）
- [ ] 支持抓取爆款帖子的评论区
- [ ] 数据自动写入飞书多维表格
- [ ] 可手动触发任务执行

### 4.3 后续迭代方向

- 增加关键词数量
- AI智能生成回复建议
- 评论情感分析（正/负向）
- 竞品账号监控
- 数据可视化看板

## 五、运营策略

### 5.1 冷启动数据积累
- 第一周：纯监控，不互动，积累数据基线
- 第二周：开始抓取评论，了解用户真实问题
- 第三周：开始轻度互动，测试回复效果

### 5.2 互动注意事项
- 每日互动数量控制在10条以内，避免触发风控
- 回复内容避免直接留微信/外链，引导私信
- 保持人设一致性，不要频繁切换说话风格

## 六、验收标准

### 6.1 功能验收

| 检查项 | 验收标准 |
|--------|----------|
| 搜索监控 | 每日自动采集，数据准确写入表格 |
| 爆款识别 | 正确标记符合标准的帖子 |
| 评论抓取 | 抓取内容完整，无乱码/截断 |
| 数据更新 | 飞书表格实时同步，无延迟 |

### 6.2 稳定性要求
- 连续7天无故障运行
- 数据采集成功率 > 95%
- 异常情况有日志记录

## 七、风险与应对

| 风险 | 可能性 | 应对方案 |
|------|--------|----------|
| 小红书风控封号 | 中 | 控制请求频率，多账号备用 |
| 接口变更 | 中 | 监控接口变化，及时适配 |
| Cookie过期 | 高 | 定期提醒更新，失效自动告警 |
| 数据量过大 | 低 | 增加去重逻辑，定期清理历史 |
